API Overview

The VeriSage Pro API provides programmatic access to domain registration, management, and billing systems. This documentation covers authentication methods, available endpoints, request/response formats, and example implementations.

Base URL

All API requests should be made to: https://api.verisagepro.com/v1

Rate Limiting

The API implements rate limiting to ensure fair usage:

  • Authenticated requests: 1000 per hour
  • Unauthenticated requests: 100 per hour

Authentication

All API requests require authentication using API keys. Include your API key in the Authorization header of each request.

API Key Authentication

Add the following header to all requests:

Authorization: Bearer YOUR_API_KEY_HERE

Obtaining Your API Key

  1. Log in to your VeriSage Pro account
  2. Navigate to Account Settings > API Access
  3. Generate a new API key or use an existing one
  4. Store your API key securely

API Endpoints

GET /domains/search Domain Availability Check

Check the availability of domain names across multiple extensions.

Query Parameters

Parameter Type Required Description
domain string Yes Domain name to check (without extension)
tlds array No List of TLDs to check (default: com,net,org)
sensitive boolean No Include sensitive domains in results

Response Example

{
  "status": "success",
  "data": [
    {
      "domain": "example.com",
      "available": true,
      "price": 12.99,
      "currency": "USD"
    },
    {
      "domain": "example.net",
      "available": false,
      "reason": "registered"
    }
  ]
}
POST /domains/register Register a New Domain

Register a new domain name with specified contact details.

Request Body

Field Type Required Description
domain string Yes Fully qualified domain name to register
period integer Yes Registration period in years (1-10)
registrant object Yes Registrant contact information
admin object Yes Administrative contact information
tech object Yes Technical contact information
billing object Yes Billing contact information

Response Example

{
  "status": "success",
  "data": {
    "domain": "newdomain.com",
    "registration_date": "2023-06-15T10:30:00Z",
    "expiration_date": "2024-06-15T10:30:00Z",
    "nameservers": ["ns1.example.com", "ns2.example.com"]
  }
}
PUT /domains/{domain}/nameservers Update Domain Nameservers

Update the nameservers for an existing domain.

Path Parameters

Parameter Type Description
domain string Fully qualified domain name

Request Body

Field Type Required Description
nameservers array Yes List of nameserver hostnames (2-13)

Response Example

{
  "status": "success",
  "message": "Nameservers updated successfully",
  "data": {
    "domain": "example.com",
    "nameservers": [
      "ns1.newhost.com",
      "ns2.newhost.com"
    ]
  }
}
DELETE /domains/{domain} Delete Domain

Request deletion of a domain (subject to registry policies).

Path Parameters

Parameter Type Description
domain string Fully qualified domain name to delete

Response Example

{
  "status": "success",
  "message": "Domain deletion request submitted",
  "data": {
    "domain": "oldsite.com",
    "request_id": "req_1234567890"
  }
}

Code Examples

Python Example

import requests

url = "https://api.verisagepro.com/v1/domains/search"
headers = {
    "Authorization": "Bearer YOUR_API_KEY"
}
params = {
    "domain": "mynewwebsite",
    "tlds": ["com", "net", "org"]
}

response = requests.get(url, headers=headers, params=params)
data = response.json()
print(data)

JavaScript Example

async function checkDomain(domain) {
  const response = await fetch(
    `https://api.verisagepro.com/v1/domains/search?domain=${domain}&tlds=com,net`, 
    {
      method: 'GET',
      headers: {
        'Authorization': 'Bearer YOUR_API_KEY'
      }
    }
  );
  
  const data = await response.json();
  console.log(data);
}

checkDomain('example');
