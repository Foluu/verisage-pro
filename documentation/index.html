VeriSage Pro — CoCCA Webhook API

Complete integration guide for the CoCCA platform team.
This API receives financial transactions from CoCCA and posts them automatically into NiRA's SAGE 200 Evolution accounting system.

Jump to Endpoints

  API Overview

VeriSage Pro is the middleware that sits between the CoCCA domain registry platform and NiRA's SAGE 200 Evolution accounting system. When a financial transaction occurs on CoCCA — a domain registration, renewal, payment, refund, or any other event — CoCCA sends the details to VeriSage Pro via an HTTPS webhook. VeriSage Pro validates the payload, stores it, and automatically posts the corresponding accounting entry to SAGE.

Direction of Integration CoCCA sends data to VeriSage Pro. VeriSage Pro does not poll or call back to CoCCA. Every time a qualifying financial event occurs on CoCCA, CoCCA must POST the transaction details to the webhook URL provided by NiRA.

Base URL

https://<nira-server-address>:4000

NiRA will provide the exact server address. All requests must use HTTPS.

Rate Limiting

  • Webhook endpoints: 200 requests per minute
  • Exceeding this returns HTTP 429 Too Many Requests
  • Implement exponential back-off on 429 responses

Idempotency

The cocca_transaction_ref field is the idempotency key. If CoCCA sends the same reference more than once, VeriSage Pro returns 200 OK with "duplicate": true and takes no further action. CoCCA may safely retry any timed-out or failed HTTP request without risk of double-posting to SAGE.

  Authentication

All requests to the webhook endpoint must include the API Key issued by NiRA in the X-API-Key request header.

Header Format

COCCA_API_KEY: <a1f0717c68e4638a178c7419a0bf6d1b084ff4028de755f01a211d74f928b5b7>
Security An API Key has been generated for you to access the VeriSage Pro API. Please keep this key secure and do not share it publicly. If you believe your API key has been compromised, please contact our support team immediately to have it revoked and a new key issued.

Authentication Error Responses

HTTP StatusConditionResponse Body
401 Unauthorized X-API-Key header missing entirely {"success":false,"error":"Unauthorized: X-API-Key header is required"}
403 Forbidden X-API-Key header present but value is incorrect {"success":false,"error":"Forbidden: Invalid API key"}

  Transaction Fields

Every transaction payload sent to VeriSage Pro must include the following fields. All required fields must be present; missing or invalid fields return 422 Unprocessable Entity with a details array identifying the problem.

Field NiRA Name Type Required Description
cocca_transaction_ref — String Required CoCCA's unique transaction reference. Used as the idempotency key. Must be globally unique across all time. Re-sending the same reference is safe — it will be acknowledged and ignored.
registrar_id — String Required CoCCA's internal registrar identifier (e.g. REG-001). Must match a registrar mapping configured in VeriSage Pro by NiRA.
registrar_name Client Name String Required Full name of the registrar as it appears in CoCCA. This is posted to SAGE as the account name on the receipt document.
transaction_type Transaction Type String Required The type of financial transaction. Must be one of the 12 accepted values listed in the Transaction Types section. Controls which SAGE document type is created (Receipt or Credit Note).
domain_name Domain Name String Required The fully qualified domain name associated with this transaction (e.g. example.ng). For non-domain financial transactions such as Bank Payment or CC Payment, send the registrar's primary domain or account identifier.
description Description String (text) Required Free-text description of the transaction. This text appears verbatim on the SAGE accounting document. Be specific — include the domain name, period, and any relevant reference numbers.
amount Amount Decimal Required Total transaction amount inclusive of tax, in Nigerian Naira (NGN). Must be a positive number greater than 0. Send as a numeric value, not a string. Decimal places are accepted (e.g. 6450.00).
vat_amount Tax Decimal Required The tax (VAT) portion of the total amount, in NGN. Send 0 for VAT-exempt transactions. Must be a non-negative number. This is posted separately to SAGE so it appears correctly in tax reports.
amount_excl_vat — Decimal Optional Amount before tax. If omitted, VeriSage Pro calculates it as amount - vat_amount. Providing it explicitly is recommended for accuracy.
top_up_date — String (ISO 8601) Required The date and time the transaction occurred on CoCCA, in UTC. Format: YYYY-MM-DDTHH:mm:ssZ. Example: 2025-06-01T10:30:00Z.
payment_method — String Optional How the payment was made. Accepted values: bank_transfer, card_payment, ussd, mobile_money, cheque, other. Included on the SAGE line item description.
package_name — String Optional Name of the service package. Example: .ng Domain Registration - 1 Year. Used in income recognition reports.
registration_years — Integer Conditional Number of years for domain registration or renewal (1–10). Required when transaction_type is Registration, Renewal, or Auto Renewal. Controls how VeriSage Pro spreads income recognition across months in SAGE.

  Transaction Types

The transaction_type field must contain exactly one of the following values. The value is case-sensitive. Any value not in this list will be rejected with a 422 validation error.

Registration Renewal Auto Renewal Restoration Fee Transfer Fee Refund Adjustment Bank Payment CC Payment Access Fee CC Fees Registrant Change Fee
Transaction TypeSAGE Document TypeRevenue Schedule Generated?Notes
Registration Receipt (DocType 2) Yes — monthly accrual Requires registration_years. Income spread over the registration period.
Renewal Receipt (DocType 2) Yes — monthly accrual Requires registration_years.
Auto Renewal Receipt (DocType 2) Yes — monthly accrual Requires registration_years. System-initiated renewal.
Restoration Fee Receipt (DocType 2) No One-time fee for restoring an expired domain.
Transfer Fee Receipt (DocType 2) Yes — monthly accrual Registrar-to-registrar transfer. Requires registration_years if a registration period is involved.
Refund Credit Note (DocType 3)No Posted to SAGE as a Credit Note, not a Receipt. Use for any transaction that returns money to the registrar.
Adjustment Receipt (DocType 2) No Balance adjustments and corrections.
Bank Payment Receipt (DocType 2) No Direct bank payment or deposit to registrar account.
CC Payment Receipt (DocType 2) No Credit card payment to registrar account.
Access Fee Receipt (DocType 2) No Registry access or platform fee charged to registrar.
CC Fees Receipt (DocType 2) No Credit card processing fees passed through to registrar.
Registrant Change FeeReceipt (DocType 2) No Fee for changing the domain registrant contact.
Important — Refunds The Refund type is the only type that creates a Credit Note in SAGE rather than a Receipt. Send a positive amount value representing the refund amount. VeriSage Pro handles the correct SAGE document type automatically. Do not send a negative amount.

  Webhook Endpoints

POST /api/webhook/topup Submit a Single Transaction

Send a single financial transaction from CoCCA to VeriSage Pro. Use this endpoint for real-time transaction forwarding.

Request Headers

HeaderValueRequired
Content-Typeapplication/jsonRequired
X-API-KeyYour NiRA-issued API keyRequired

Request Body

See the Fields section above for full descriptions of each field.

{
  "cocca_transaction_ref": "TXN-2025-001234",
  "registrar_id":          "REG-001",
  "registrar_name":        "Acme Domains Ltd",
  "transaction_type":      "Registration",
  "domain_name":           "example.ng",
  "description":           "New .ng domain registration for example.ng — 1 year",
  "amount":                6450.00,
  "vat_amount":            450.00,
  "amount_excl_vat":       6000.00,
  "payment_method":        "bank_transfer",
  "top_up_date":           "2025-06-01T10:30:00Z",
  "package_name":          ".ng Domain Registration — 1 Year",
  "registration_years":    1
}

Success Response — 202 Accepted

Returned when the transaction has been received and queued for SAGE posting.

{
  "success":  true,
  "message":  "Transaction received and queued for SAGE posting",
  "id":       42,
  "coccaRef": "TXN-2025-001234"
}

Duplicate Response — 200 OK

Returned when the same cocca_transaction_ref has already been received. Safe to ignore.

{
  "success":   true,
  "message":   "Duplicate transaction — already processed",
  "duplicate": true,
  "id":        42
}
POST /api/webhook/topup/batch Submit Multiple Transactions

Send multiple transactions in a single HTTP request. Use this for bulk sync, catch-up after downtime, or historical imports. Maximum recommended batch size: 100 transactions per request.

Request Body

{
  "transactions": [
    {
      "cocca_transaction_ref": "TXN-2025-001240",
      "registrar_id":          "REG-002",
      "registrar_name":        "NetBridge Communications Nigeria",
      "transaction_type":      "Renewal",
      "domain_name":           "netbridge.com.ng",
      "description":           "Annual renewal for netbridge.com.ng — 2 years",
      "amount":                12900.00,
      "vat_amount":            900.00,
      "amount_excl_vat":       12000.00,
      "top_up_date":           "2025-06-01T09:00:00Z",
      "registration_years":    2
    },
    {
      "cocca_transaction_ref": "TXN-2025-001241",
      "registrar_id":          "REG-001",
      "registrar_name":        "Acme Domains Ltd",
      "transaction_type":      "Refund",
      "domain_name":           "acme-old.ng",
      "description":           "Refund for cancelled registration of acme-old.ng",
      "amount":                6450.00,
      "vat_amount":            450.00,
      "amount_excl_vat":       6000.00,
      "top_up_date":           "2025-06-01T09:05:00Z"
    }
  ]
}

Response — 202 Accepted

Each transaction in the batch is processed independently. Check each result's status field.

{
  "success":   true,
  "processed": 2,
  "results": [
    { "ref": "TXN-2025-001240", "status": "accepted", "duplicate": false },
    { "ref": "TXN-2025-001241", "status": "accepted", "duplicate": false }
  ]
}
GET /health System Health Check

No authentication required. Call this to verify VeriSage Pro is online before sending transactions. The response also shows whether SAGE and the VeriSage database are connected. 

{
  "status":    "ok",
  "service":   "VeriSage Pro",
  "timestamp": "2025-06-01T10:00:00.000Z",
  "database": {
    "appDb":  { "connected": true, "error": null },
    "sageDb": { "connected": true, "error": null }
  }
}

  Error Responses

All error responses follow the same JSON structure:

{
  "success": false,
  "error":   "Human-readable error message",
  "details": [ ... ]  // present only on 422 validation errors
}
HTTP StatusWhen It OccursAction Required
200 OK Duplicate cocca_transaction_ref re-submitted No action. Transaction was already received successfully.
202 Accepted Transaction received and queued for SAGE posting No action. Posting to SAGE is automatic.
401 Unauthorized X-API-Key header missing Add the X-API-Key header to every request.
403 Forbidden X-API-Key value is incorrect Check the key value. Contact NiRA if correct key is unknown.
422 Unprocessable Entity A required field is missing, the wrong type, or transaction_type is not in the accepted list Read the details array. Fix the identified field(s) and retry.
429 Too Many Requests More than 200 requests per minute Implement exponential back-off. Reduce request frequency.
500 Internal Server Error Unexpected error on the VeriSage Pro server Log the request body and retry after 60 seconds. Alert NiRA if it persists.

Validation Error Example (422)

{
  "success": false,
  "error":   "Validation failed",
  "details": [
    {
      "type":     "field",
      "msg":      "transaction_type must be one of: Registration, Renewal, ...",
      "path":     "transaction_type",
      "location": "body"
    }
  ]
}
